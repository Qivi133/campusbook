## 1. 项目概述

### 1.1 项目背景

随着高校教材价格的持续上涨，大学生对于二手书籍的需求日益增长。校园内存在大量的闲置教材和专业书籍，这些书籍对于下一届学生来说仍然具有很高的使用价值。然而，传统的二手书交易方式（如校园公告栏、QQ群等）存在信息分散、交易效率低、缺乏保障等问题。

CampusBook 旨在为大学生提供一个安全、便捷、高效的二手书交易平台，帮助学生快速找到所需的二手书籍，同时让闲置书籍发挥最大价值。

### 1.2 项目目标

1. **降低学习成本**：通过二手书交易，帮助学生节省教材购买费用
2. **提高交易效率**：提供便捷的搜索和筛选功能，快速匹配供需信息
3. **保障交易安全**：建立用户评价体系，提供交易保障
4. **促进资源循环**：推动校园书籍资源的循环利用，减少浪费
5. **构建社区氛围**：打造校园文化交流平台，促进学生间的互动

### 1.3 项目范围

| 类别 | 范围 |
|------|------|
| 功能范围 | 用户管理、书籍发布与浏览、搜索筛选、购物车、订单管理、消息通讯 |
| 用户范围 | 高校在校学生 |
| 地域范围 | 单校园部署 |
| 平台范围 | Web端（响应式设计，支持移动端） |

### 1.4 关键术语

| 术语 | 定义 |
|------|------|
| 在售书籍 | 状态为"available"的书籍，可以被购买 |
| 已售书籍 | 状态为"sold"的书籍，已完成交易 |
| 购物车 | 用户临时存放欲购买商品的容器 |
| 订单 | 买家与卖家之间达成的交易协议 |
| 消息 | 用户之间关于特定书籍的沟通记录 |

## 2. 功能需求

### 2.1 用户系统

#### 2.1.1 用户注册

**功能描述**：新用户通过填写必要信息创建账户

**业务流程**：
1. 用户访问注册页面
2. 填写用户名、邮箱、密码、手机号（可选）
3. 系统验证信息合法性
4. 系统检查用户名和邮箱是否已存在
5. 创建用户账户
6. 自动登录并跳转至首页

**输入要求**：
| 字段 | 类型 | 必填 | 验证规则 |
|------|------|------|----------|
| 用户名 | 字符串 | 是 | 2-50字符，唯一 |
| 邮箱 | 邮箱格式 | 是 | 有效邮箱格式，唯一 |
| 密码 | 字符串 | 是 | 至少6位 |
| 手机号 | 字符串 | 否 | 11位数字（可选） |

**输出结果**：
- 成功：跳转首页，显示"注册成功"提示
- 失败：显示具体错误信息

**业务规则**：
- 用户名不能包含敏感词
- 邮箱不能重复注册
- 密码使用哈希存储，不明文保存

#### 2.1.2 用户登录

**功能描述**：已注册用户登录系统

**业务流程**：
1. 用户访问登录页面
2. 输入用户名和密码
3. 系统验证凭证
4. 登录成功，创建会话
5. 跳转至首页或之前访问的页面

**输入要求**：
| 字段 | 类型 | 必填 |
|------|------|------|
| 用户名 | 字符串 | 是 |
| 密码 | 字符串 | 是 |

**输出结果**：
- 成功：跳转首页，显示欢迎信息
- 失败：显示"用户名或密码错误"

**业务规则**：
- 登录成功后创建Session
- 登录状态通过Cookie保持

#### 2.1.3 用户登出

**功能描述**：结束当前用户会话

**业务流程**：
1. 用户点击"登出"按钮
2. 清除用户Session
3. 跳转至首页
4. 显示"已成功登出"提示

#### 2.1.4 个人资料管理

**功能描述**：用户查看和修改个人信息

**业务流程**：
1. 用户访问个人资料页面
2. 查看当前个人信息
3. 修改所需信息
4. 提交保存
5. 系统更新数据库
6. 显示"保存成功"提示

**可修改字段**：
| 字段 | 说明 |
|------|------|
| 用户名 | 用户昵称 |
| 邮箱 | 联系邮箱 |
| 手机号 | 联系电话 |
| 密码 | 需输入原密码验证 |

**业务规则**：
- 修改邮箱时需验证格式
- 修改密码需要验证原密码
- 用户名和邮箱仍需保持唯一性

#### 2.1.5 用户统计信息

**功能描述**：展示用户的活动统计数据

**显示内容**：
- 发布的书籍数量
- 购买订单数量
- 消息总数

### 2.2 书籍系统

#### 2.2.1 发布书籍

**功能描述**：卖家发布待售二手书信息

**业务流程**：
1. 登录用户点击"发布书籍"
2. 填写书籍信息
3. 上传书籍图片（可选）
4. 提交发布
5. 系统创建书籍记录
6. 跳转至书籍详情页

**输入要求**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 书名 | 字符串 | 是 | 最多200字符 |
| 作者 | 字符串 | 否 | 书籍作者 |
| ISBN | 字符串 | 否 | 书籍ISBN号 |
| 分类 | 下拉选择 | 是 | 预设分类之一 |
| 价格 | 数字 | 是 | 单位：元 |
| 新旧程度 | 下拉选择 | 否 | 预设选项 |
| 描述 | 文本 | 否 | 书籍情况描述 |
| 图片 | 文件 | 否 | 支持jpg/png/gif |

**预设分类**：
- 教材
- 文学
- 专业课
- 考研
- 公务员
- 其他

**新旧程度选项**：
- 全新
- 九成新
- 八成新
- 七成新
- 六成新及以下

**业务规则**：
- 书籍默认状态为"available"（在售）
- 图片保存至服务器本地存储
- 发布者自动关联当前登录用户

#### 2.2.2 浏览书籍

**功能描述**：用户查看所有在售书籍列表

**展示内容**：
- 书籍封面图片
- 书名
- 作者
- 价格
- 分类标签
- 状态标签（在售/已售）

**分页规则**：
- 每页显示12本书
- 支持页码导航

**排序方式**：
- 最新发布（默认）
- 价格从低到高
- 价格从高到低

#### 2.2.3 书籍详情

**功能描述**：查看单本书籍的完整信息

**展示内容**：
| 内容 | 说明 |
|------|------|
| 书籍图片 | 封面大图 |
| 书名 | 书籍标题 |
| 作者 | 书籍作者 |
| 价格 | 售价 |
| 分类 | 所属分类 |
| ISBN | 书籍ISBN |
| 新旧程度 | 书籍成色 |
| 卖家 | 发布者用户名 |
| 发布时间 | 上架时间 |
| 描述 | 详细说明 |

**页面功能**：
| 用户角色 | 可用操作 |
|----------|----------|
| 游客 | 查看信息 |
| 卖家本人 | 编辑、删除 |
| 买家 | 加入购物车、联系卖家 |

#### 2.2.4 编辑书籍

**功能描述**：卖家修改已发布书籍的信息

**业务流程**：
1. 书籍所有者访问编辑页面
2. 修改书籍信息
3. 上传新图片（可选）
4. 保存修改
5. 更新数据库
6. 跳转至书籍详情页

**业务规则**：
- 仅书籍所有者可编辑
- 已售书籍不允许编辑

#### 2.2.5 删除书籍

**功能描述**：卖家删除已发布的书籍

**业务流程**：
1. 书籍所有者点击删除
2. 弹出确认对话框
3. 用户确认
4. 删除书籍记录
5. 同时删除相关购物车记录
6. 跳转至首页

**业务规则**：
- 仅书籍所有者可删除
- 删除后不可恢复
- 关联的购物车记录一并清除

#### 2.2.6 我的书籍

**功能描述**：查看自己发布的所有书籍

**展示内容**：
- 发布的书籍列表
- 书籍状态（在售/已售）
- 快速链接至详情页

### 2.3 搜索系统

#### 2.3.1 关键词搜索

**功能描述**：通过关键字查找书籍

**搜索范围**：
- 书名
- 作者

**输入规则**：
- 支持模糊匹配
- 不区分大小写

#### 2.3.2 分类筛选

**功能描述**：按分类快速筛选书籍

**筛选方式**：
- 选择一个分类
- 显示该分类下的所有在售书籍
- 可与其他筛选条件组合

#### 2.3.3 价格区间筛选

**功能描述**：按价格范围筛选书籍

**输入规则**：
- 最低价格（可为空）
- 最高价格（可为空）
- 价格单位：元

#### 2.3.4 排序功能

**功能描述**：按指定方式排序书籍列表

**排序选项**：
| 选项 | 说明 |
|------|------|
| 最新发布 | 按发布时间倒序 |
| 价格从低到高 | 价格升序 |
| 价格从高到低 | 价格降序 |

### 2.4 购物车系统

#### 2.4.1 加入购物车

**功能描述**：买家将心仪的书籍加入购物车

**业务流程**：
1. 买家在书籍详情页点击"加入购物车"
2. 系统验证书籍状态
3. 检查是否已加入
4. 创建购物车记录
5. 跳转至购物车页面

**业务规则**：
- 仅登录用户可使用
- 不能购买自己的书籍
- 书籍必须在售状态
- 不能重复添加同一本书

#### 2.4.2 查看购物车

**功能描述**：查看已添加的所有书籍

**展示内容**：
| 内容 | 说明 |
|------|------|
| 书籍图片 | 缩略图 |
| 书名 | 链接至详情 |
| 卖家 | 出售者 |
| 价格 | 单价 |
| 状态 | 在售/已下架 |
| 操作 | 移除按钮 |

**统计信息**：
- 购物车总数量
- 购物车总价合计

#### 2.4.3 移出购物车

**功能描述**：从购物车中移除书籍

**业务流程**：
1. 用户点击"移除"
2. 删除购物车记录
3. 刷新购物车页面
4. 显示"已移除"提示

---

### 2.5 订单系统

#### 2.5.1 创建订单

**功能描述**：买家将购物车中的书籍下单购买

**业务流程**：
1. 用户在购物车页面点击"立即下单"
2. 填写给卖家的留言（可选）
3. 确认下单
4. 为每个在售书籍创建订单
5. 书籍状态更新为"已售"
6. 清空购物车
7. 跳转至订单页面
8. 显示"订单创建成功"

**业务规则**：
- 购物车为空时不能下单
- 只会为在售书籍创建订单
- 已下架书籍不会生成订单

#### 2.5.2 订单列表

**功能描述**：查看购买和出售的订单

**购买订单展示**：
| 内容 | 说明 |
|------|------|
| 书籍信息 | 图片、书名 |
| 卖家 | 出售者用户名 |
| 价格 | 订单金额 |
| 状态 | 待确认/已完成/已取消 |
| 时间 | 下单时间 |
| 留言 | 给卖家的留言 |

**出售订单展示**：
| 内容 | 说明 |
|------|------|
| 书籍信息 | 图片、书名 |
| 买家 | 购买者用户名 |
| 价格 | 售出金额 |
| 状态 | 交易状态 |
| 时间 | 售出时间 |

**订单状态**：
| 状态 | 说明 |
|------|------|
| pending | 待确认（已下单未收货） |
| completed | 已完成（已确认收货） |
| cancelled | 已取消 |

#### 2.5.3 确认收货

**功能描述**：买家确认收到商品

**业务流程**：
1. 买家点击"确认收货"
2. 更新订单状态为"completed"
3. 刷新订单页面
4. 显示"已确认收货"

**业务规则**：
- 仅买家可操作
- 仅pending状态订单可确认

#### 2.5.4 取消订单

**功能描述**：买家取消未完成的订单

**业务流程**：
1. 买家点击"取消订单"
2. 弹出确认对话框
3. 用户确认
4. 订单状态更新为"cancelled"
5. 书籍状态恢复为"available"
6. 刷新订单页面

**业务规则**：
- 仅买家可操作
- 仅pending状态订单可取消
- 取消后书籍重新变为可售状态

- 仅消息接收者可标记

## 4. 数据库模型

### 4.1 ER图关系

```
User (1) ----< Book
   |              |
   |              +--< Order
   |
   +--< CartItem
   |
   +--< Message
```

### 4.2 数据表

#### User (用户)
| 字段 | 类型 | 说明 |

| id | Integer | 主键 |

| created_at | DateTime | 创建时间 |

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| title | String(200) | 书名 |
| author | String(100) | 作者 |
| isbn | String(20) | ISBN |
| category | String(50) | 分类 |
| price | Float | 价格 |
| description | Text | 描述 |
| image_url | String(200) | 图片URL |
| condition | String(20) | 新旧程度 |
| seller_id | Integer | 卖家ID |
| status | String(20) | 状态 (available/sold) |
| created_at | DateTime | 发布时间 |

#### Order (订单)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| buyer_id | Integer | 买家ID |
| book_id | Integer | 书籍ID |
| total_price | Float | 总价 |
| status | String(20) | 状态 |
| contact_message | String(500) | 留言 |

#### CartItem (购物车)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| user_id | Integer | 用户ID |
| book_id | Integer | 书籍ID |

#### Message (消息)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| sender_id | Integer | 发送者ID |
| receiver_id | Integer | 接收者ID |
| content | String(500) | 内容 |
| book_id | Integer | 相关书籍ID |
| created_at | DateTime | 发送时间 |
| is_read | Boolean | 已读状态 |

## 5. 验收标准

### 5.1 功能验收

| 序号 | 功能 | 验收条件 |
|------|------|----------|
| 1 | 用户注册 | 可成功创建账户，重复用户名/邮箱提示错误 |
| 2 | 用户登录 | 正确凭证可登录，错误凭证提示失败 |
| 3 | 发布书籍 | 填写必填项后可成功发布，图片可上传 |
